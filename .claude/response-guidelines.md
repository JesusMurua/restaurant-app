# Response Guidelines for Development Assistance

## Core Principles
1. **Follow exact instructions** - Only implement what is explicitly requested, nothing more.
2. **Step-by-step approach** - Wait for confirmation between implementation steps.
3. **Analysis before coding** - Evaluate the situation based on context before writing any code.
4. **Best practices focus** - Consider coding standards and best practices for each implementation.
5. **Feasibility check** - Assess if the requested implementation is viable before proceeding.
6. **Code language standard** - All code, comments, method names, variables, and technical documentation must be in English.

## Process Flow
1. When receiving a request, first **analyze the context** and provide thoughts on:
   - How the request fits into existing code structure
   - Potential implementation approaches
   - Any concerns or considerations

2. **Wait for confirmation** before implementing any solution.

3. When approved to proceed, implement **only the specific code requested**.

4. Do not add:
   - Additional methods not requested
   - Extra properties
   - Styling changes
   - Functionality beyond the scope of the request

5. After implementation, summarize what was done and ask if any adjustments are needed.

## Documentation Standards
- All code comments should follow JSDoc format
- Use region tags to organize code as per coding standards
- Maintain consistent naming conventions as shown in the coding standards
## Language Guidelines
- **Code**: All code, comments, method names, variables, and documentation (including JSDoc) must be written in English
- **Technical terms**: Technical terms should always be maintained in English
- **Chat interaction**:
  * Casual explanations and general conversation should be in Spanish
  * Technical analysis can be provided in both Spanish and English when needed
  * Questions about code should be answered with explanations in Spanish, but code examples in English

## Implementation Standards
- Code should be written according to the Angular coding standards document
- Follow observable management practices as outlined
- Implement proper error handling
- Use type safety throughout
- Organize complex components according to the standards
